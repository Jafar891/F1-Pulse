# F1-Pulse
🏎️ F1-Pulse: End-to-End Data LakehouseAutomated F1 Telemetry & Performance Pipeline on Databricks📋 Project OverviewF1-Pulse is a production-grade data engineering pipeline that ingests, transforms, and analyzes Formula 1 racing data. Using the Medallion Architecture, the project converts raw JSON responses from the OpenF1 API into actionable driver performance insights.The pipeline is currently configured to process the 2025 Abu Dhabi Grand Prix, demonstrating the ability to handle complex joins, sensor telemetry, and historical race analysis.🛠️ Tech StackPlatform: Databricks (Serverless Compute)Language: Python (PySpark, Pandas)Storage: Delta Lake (ACID Transactions, Time Travel)Governance: Unity Catalog (3-layer Namespace)Version Control: GitHub IntegrationOrchestration: Databricks Workflows (Jobs)🏗️ Architecture Layers1. Bronze (Raw Ingestion)Source: REST API (OpenF1)Method: Python requests with Pandas-to-Spark conversion for type safety.Storage: Raw Delta tables for Sessions, Laps, Drivers, and Telemetry.Audit: Added ingested_at timestamps to all raw records.2. Silver (Cleaned & Enriched)Transformation: Schema enforcement and type casting (converting strings to timestamps, floats, and booleans).Enrichment: Performed inner joins between Laps and Drivers tables to map driver numbers to full names and constructor teams.Quality: Filtered out "Pit Out" laps to ensure timing analysis remained accurate.3. Gold (Aggregated Insights)Analytics: Aggregated session data to calculate Fastest Lap, Average Lap Pace, and Total Valid Laps per driver.Result: A dashboard-ready leaderboard showcasing performance metrics across the 2025 grid.📈 Sample Results (2025 Abu Dhabi GP)TeamDriverFastest Lap (s)Avg Lap Time (s)FerrariCharles LECLERC86.7288.79McLarenOscar PIASTRI86.7688.98Red Bull RacingMax VERSTAPPEN87.6288.75MercedesKimi ANTONELLI88.0290.20🚀 Key Engineering FeaturesCI/CD Integration: Notebooks are version-controlled and synced directly from this GitHub repository.Idempotent Pipelines: Uses overwrite logic to ensure re-running the pipeline does not create duplicate data.Error Handling: Implemented "Safe Ingestion" patterns to manage null values and undetermined schema types from the API.Automation: Managed by Databricks Workflows, allowing for scheduled daily or weekly updates.📂 Repository Structure01_Ingest_F1_Data: Connects to API and populates Bronze layer.02_Silver_Transformation: Cleans, types, and joins data into Silver layer.03_Gold_Analytics: Generates final driver performance metrics for the Gold layer.
